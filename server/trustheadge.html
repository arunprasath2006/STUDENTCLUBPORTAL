THE BIGGEST THREAT TO ENTERPRISE DATA IS NOT HACKERS — IT IS INSIDERS
The Problem

Every company has a trust gap. Nobody is watching it.

Inside every enterprise, employees access sensitive data hundreds of times a day — financial records, patient files, payroll data. Most of that access is legitimate. Some of it isn't. The terrifying part? Nobody knows the difference.

Traditional systems record events but never analyse them. Logs pile up in a database that the very admin you're trying to audit can edit or delete. When a breach happens, the answer to "who accessed this?" is almost always: we don't know.

This is the Trust Gap. It exists in every hospital, bank, and enterprise. It is legal liability and compliance nightmare — sitting silently, waiting.

The Idea

TrustLedger: The Black Box for Enterprise Data

"Like an airplane's flight recorder — but for every sensitive record your employees ever touch."
Core Concept · Samhita Hackathon 2026

TrustLedger is a security intelligence layer that wraps around any enterprise system. Every data access — who, what, when, from where — is captured, cryptographically chained, and fed into an AI engine that watches for patterns no human could detect manually.

The result: a tamper-proof record of everything that happened to your sensitive data, with an AI that raises the alarm the moment behaviour crosses the line — automatically, in real time, before damage is done.

Three Properties That Win
  • 1.Immutable chain — no admin can erase evidence
  • 2.AI anomaly scoring — patterns humans miss, caught instantly
  • 3.PII-anonymized logs — audit without creating a second leak
How It Works

Three Engines. One System.

① Controlled Access Layer

All sensitive data is accessed through a single authenticated API gateway. JWT tokens enforce who can access what. Every query passes through TrustLedger first — no direct DB access ever.

② Tamper-Proof Audit Chain

Every access event is written as an immutable log entry, chained to the previous via SHA-256. Edit any entry and every subsequent hash breaks — mathematical proof of tampering.

# Every access writes one entry user: analyst_24 (pseudonymized) action: READ financial_record #8821 time: 2026-03-07 10:41:33 prev_hash: 9a12cd4f8b... curr_hash: 8fd21ab7c3... ← breaks if tampered

③ AI Anomaly Detection

An Isolation Forest model trains on baseline patterns. Access frequency, time of day, data type, volume — all become features. Deviations score a risk level. No human needed.
